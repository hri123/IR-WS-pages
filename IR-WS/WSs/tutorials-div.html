Summary
  • learning spring and hibernate that may be difficult to be deployed over OSGi can be done here
Setup
  1. Eclipse -> Install new software -> m2e (if you are using eclipse 3.6, you need to use m2e 1.3 version, the later versions of m2e is only for juno and above) (dont use the m2eclipse command line tools, i.e. mvn eclipse:eclipse)
  2. Now clone (copy) the files into your workspace (the m2e plugin will not copy it into your workspace when you import)
  3. Import -> Maven -> existing projects into workspace and finish (importing as Maven project is important here)
  4. Some issues with the older versions of m2e looks like. The .classpath file auto-generated has some issues. excluding="**/*.java" attribute must be for the src/main/resources and including="**/*.java" attribute must be for the src/main/java and similarly for test
  5. But the important problem is the entry - must be added under else you will get - java.lang.ClassNotFoundException: org.springframework.web.servlet.DispatcherServlet
  6. restart eclipse after the classpath changes
  7. Database: Currently uses DB2, check the viralpatel.net links for DDL to create the DB table. Can be used with Derby or something.
  8. Copy db2jcc.jar, db2jcc_license_cisuz.jar, db2jcc_license_cu.jar into .../apache-tomcat-6.0.20/lib; in case of derby copy derby.jar into the apache-tomcat lib directory;
  9. To execute: Add new server environment - Tomcat, Add new server
